# 新手操作指南

这份指南按“体检 → 启动 → 图片 → 编辑 → 视频”的顺序操作。第一次只跑 2 秒测试，不要直接生成长视频。

## 一、首次安装

准备条件：Windows、RX 9070 XT 16GB、32GB 内存、Python 3.12、Git，以及至少 150GB 可用磁盘空间。建议 Windows 页面文件保留足够自动扩展空间。

在 PowerShell 中运行：

```powershell
git clone https://github.com/Justin-Ju-0413/comfyui-amd-rocm-kit.git
cd comfyui-amd-rocm-kit
powershell -ExecutionPolicy Bypass -File .\scripts\bootstrap.ps1
```

安装图片模型：

```powershell
.\scripts\download-models.ps1 -Group image-preview,image-common
```

需要高质量图片时再下载：

```powershell
.\scripts\download-models.ps1 -Group image-quality
```

需要视频时再下载：

```powershell
.\scripts\download-models.ps1 -Group video,video-audio
```

## 二、一键体检

```powershell
.\scripts\doctor.ps1
```

通过时应看到 RX 9070 XT、PyTorch `2.9.1+rocm7.2.1`、约 16GB 显存和 `pip check` 成功。黄色的“可选模型缺失”可以按需处理；红色错误必须先修复。

## 三、启动 ComfyUI

双击 `scripts\start-image.bat`，或在命令行运行：

```powershell
.\scripts\start-image.bat
```

视频也使用 NORMAL_VRAM：

```powershell
.\scripts\start-video.bat
```

打开 <http://127.0.0.1:8188>。同一时间只启动一个 8188 服务。

## 四、生成第一张图片

1. 把 `workflows/image/flux2_klein_4b_preview.ui.json` 拖进 ComfyUI 页面。
2. 找到提示词节点，输入想要的画面描述。
3. 保持默认 4 步、CFG 1、Euler。
4. 点击 Queue/运行。
5. 图片保存在 `<ComfyUI目录>\output\kit`。

质量优先时加载 `flux2_klein_9b_quality.ui.json`。9B 首次加载更慢，不要同时运行视频任务。

## 五、自然语言编辑图片

API 示例 `workflows/image/flux2_klein_4b_edit.api.json` 使用正确的 `VAEEncode → ReferenceLatent` 链。

1. 把待编辑图片复制到 `<ComfyUI目录>\input`。
2. 将工作流中的 `kit_test_portrait.png` 改成你的文件名。
3. 修改编辑提示词，例如“保持人物身份，把背景换成夜晚城市”。
4. 运行后检查主体身份和未编辑区域。

如需局部编辑，先运行 `workflows/auxiliary/sam1_subject_mask.api.json` 获取蒙版，再把蒙版接入编辑工作流。

## 六、生成第一段视频

第一次使用 `workflows/video/ltx23_i2v_2s_silent.api.json`：

1. 把起始图放入 `<ComfyUI目录>\input`。
2. 修改 `LoadImage` 文件名。
3. 使用动作明确、镜头简单的提示词。
4. 提交后耐心等待。首次模型装载会大量占用内存，短时接近满内存属于已知现象。
5. 输出位于 `<ComfyUI目录>\output\kit`。

无声版成功后再运行 `ltx23_i2v_2s_audio.api.json`。最后才尝试 `ltx23_i2v_5s_audio.api.json`。

不要把 `--lowvram` 当作默认视频模式；本机实测会导致 22B 模型在 CPU/GPU 间频繁搬运，速度极慢。

## 七、完整“编辑 → 视频”

1. 运行 Flux 编辑工作流。
2. 把选中的编辑结果复制到输入目录并命名为 `kit_edited_input.png`。
3. 运行 `workflows/video/flux_edit_to_ltx23.api.json`。
4. 根据编辑后的主体重写动作提示词。

## 八、停止任务与恢复

- 正在采样时点击 ComfyUI 的 Cancel/Interrupt。
- 队列异常时先清空 pending queue，不要反复点击运行。
- 浏览器无响应但服务仍在时刷新页面，输出不会因此删除。
- ROCm 无法启动时才使用 `start-directml-fallback.bat`；该模式速度和显存识别不作为正式基线。

## 九、常见问题

### 模型名称变红

模型未下载或目录错误。运行 `doctor.ps1`，按清单把文件放到指定相对路径，然后重启 ComfyUI。

### LTX 解码 OOM

确认 VAE tile 为 256、分辨率为 512×704，并关闭其他 GPU 程序。不要把 tile 改回 512。

### DWPose 出黑图

使用本仓库启动脚本，它会设置 `AUX_ORT_PROVIDERS=CPUExecutionProvider`。本机 DirectML ONNX 结果不正确。

### SAM2 出现棋盘纹

使用 SAM1 ViT-B 和仓库提供的阈值、膨胀、收缩后处理链。

### 视频动作太小

减少镜头描述，明确写出单一动作和时间顺序，例如“抬起双臂、左右转身、最后停在正面”。
